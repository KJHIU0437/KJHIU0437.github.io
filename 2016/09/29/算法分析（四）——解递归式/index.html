世界的樣子取決於寧的態度
算法分析（四）——解递归式
发布于：2016-09-29 15:54:47 标签：/ Algorithms / 访问：

在前一篇文章中，我们讨论了循环的分析。许多算法本质上是递归的。当我们分析它们时，我们得到了时间复杂度的递归关系。我们得到运行时间对于大小为n的输入是n的函数，对于较小的输入是运行时间。例如，在归并排序中，为了对给定的数组进行排序，我们将它分成两半，然后递归地重复这一过程。最后我们合并结果。归并排序的时间复杂度可以写成T(n) = 2T(n/2) + cn。还有许多其他的算法，如二分查找、汉诺塔等。

解递归主要有三种方法。

1)代换法

先对解进行猜测，然后用数学归纳法来证明猜测的正确与否。

2)递归树法

在此方法中，我们绘制递归树，计算树的每一层所花费的时间。最后，我们总结了各级所做的工作。为了绘制递归树，我们从给定的递归树开始，不断地绘制，直到找到层次之间的模式。模式通常是一个算术或几何级数。

3)套用公式法:

主方法是求解的直接方法。主方法只适用于以下类型的递归或可转换为以下类型的递归。

有以下三种情况:

  1. If f(n) = Θ(nc) where c < Logba then T(n) = Θ(nLogba)

  2. If f(n) = Θ(nc) where c = Logba then T(n) = Θ(ncLog n)

  3. If f(n) = Θ(nc) where c > Logba then T(n) = Θ(f(n))

这是怎么做到的呢?

套用公式法主要来源于递归树法。如果我们画递归树的T (n) = (n / b) + f (n),我们可以看到,在根所做的工作是f (n)和工作在所有的树叶都是Θc Logba (nc)。递归树的高度是Logbn

在递归树法中，我们计算所做的总功。如果工作在叶子多项式,然后叶子是占主导地位的一部分,我们的结果成为了工作在叶子(情况1)。如果工作在叶子和根是一样渐近,那么结果就变得高度乘以各级工作(例2)。如果工作根多是渐近,那么结果就是工作在根(例3)。

一些标准算法的例子，它们的时间复杂度可以用套用公式法来评估

归并排序:T (n) = 2 T (n / 2) +Θ(n)。它落在情形2中，因为c是1,Logba也是1。因此,解决方案是Θ(n Logn)

二进制搜索:T (n) = T (n / 2) +Θ(1)。当c等于0 Logba也等于0时，它也落在情形2中。所以解决方法是Θ(Logn)

注意

1) 用主定理求解T(n) = aT(n/b) + f(n)的递归式是不必要的。给定的三种情况之间有一些差距。例如，递归T(n) = 2T(n/2) + n/Logn不能用主方法求解。

2) 第二种情况可以扩展f (n) =Θ(ncLogkn)
如果f (n) =Θ(ncLogkn)常数k > = 0和c = Logba,然后T (n) =Θ(ncLogk + 1 n)
